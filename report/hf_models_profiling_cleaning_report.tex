\section{Introduction}

This report documents the ingestion, profiling, and cleaning workflow for the Hugging Face Hub model metadata used in the OSS Pulse project. The objective is to convert raw model metadata from the Hugging Face Hub API into a structured, analysis-ready dataset that can support downstream Spark analytics on AI open-source project adoption and health.

% ──────────────────────────────────────────
\section{Data Source Description}

\begin{itemize}
  \item \textbf{Data source:} Hugging Face Hub --- model metadata
  \item \textbf{Owner / access:} Public API via the \texttt{huggingface\_hub} Python library
  \item \textbf{Original format:} JSON Lines (one record per model), fetched from \texttt{HfApi.list\_models()}
  \item \textbf{Collection scope:} All public models on the Hugging Face Hub (snapshot taken on 2026-04-23)
  \item \textbf{Size:} 2,814,955 models, approximately 1.6\,GB on disk as JSONL
\end{itemize}

\section{Ingestion Workflow}

The ingestion is split into two stages.

\subsection{Stage 1: API Fetch (Pure Python)}

\begin{itemize}
  \item Script: \texttt{ingestion/ingest\_hf\_models.py} (mode: \texttt{fetch})
  \item Uses \texttt{huggingface\_hub.HfApi().list\_models(full=True, cardData=True)} to iterate over all public models
  \item Serializes each model as one JSON line into a \texttt{.jsonl} file
  \item This stage does \textbf{not} use Spark --- it is a pure Python data collection step
\end{itemize}

\subsection{Stage 2: Spark Ingestion (JSON $\rightarrow$ Parquet)}

\begin{itemize}
  \item Script: \texttt{ingestion/ingest\_hf\_models.py} (mode: \texttt{spark})
  \item Spark API: DataFrame API
  \item Input format: JSON Lines
  \item Explicit schema: \texttt{schemas/hf\_models\_schema.py}
  \item Output format: raw Parquet at \texttt{data/raw/huggingface\_hub}
\end{itemize}

\begin{lstlisting}[language=bash,caption={Ingestion commands}]
# Stage 1: fetch from API
python -m ingestion.ingest_hf_models fetch \
  --output data/source/huggingface_hub/hf_models.jsonl

# Stage 2: convert to Parquet
python -m ingestion.ingest_hf_models spark \
  --input data/source/huggingface_hub/hf_models.jsonl
\end{lstlisting}

\subsection{Observed Data Issues}

\begin{itemize}
  \item 68.3\% of models lack a \texttt{pipeline\_tag} --- they are uploaded without specifying a task type.
  \item \texttt{library\_name} is null for 53.1\% of models not associated with a recognized framework.
  \item \texttt{card\_data.license} is inconsistently formatted (e.g.\ \texttt{apache-2.0} vs \texttt{Apache 2.0}).
  \item \texttt{card\_data.datasets} is null for 94.0\% of models --- most uploaders do not specify training data.
  \item Many models have \texttt{downloads = 0} and \texttt{likes = 0}, indicating placeholder or unused uploads (median downloads is 0).
  \item \texttt{safetensors.parameters.total} is only available for models stored in safetensors format.
\end{itemize}

\section{Data Cleaning}

Script: \texttt{cleaning/clean\_hf\_models.py}

\subsection{Cleaning Steps}

\begin{enumerate}
  \item \textbf{Filter private and disabled models} --- remove records where \texttt{private = true} or \texttt{disabled = true}.
  \item \textbf{Split \texttt{modelId}} into \texttt{owner} and \texttt{model\_name} columns.
  \item \textbf{Normalize \texttt{pipeline\_tag}} --- lowercase; null values replaced with \texttt{"unknown"}.
  \item \textbf{Normalize \texttt{library\_name}} --- lowercase; null values replaced with \texttt{"unknown"}.
  \item \textbf{Parse timestamps} --- convert \texttt{lastModified} $\rightarrow$ \texttt{last\_modified\_ts} + \texttt{last\_modified\_date}; convert \texttt{created\_at} $\rightarrow$ \texttt{created\_ts}.
  \item \textbf{Extract \texttt{card\_data} fields} --- \texttt{license}, \texttt{training\_datasets}, \texttt{languages}, \texttt{base\_model}.
  \item \textbf{Normalize license} --- map common synonyms (e.g.\ \texttt{"apache 2.0"} $\rightarrow$ \texttt{"apache-2.0"}); null $\rightarrow$ \texttt{"unknown"}.
  \item \textbf{Compute derived fields} --- \texttt{tag\_count} from tags array; \texttt{parameter\_count} from safetensors.
  \item \textbf{Filter bad data} --- drop records where \texttt{modelId} is null, does not contain \texttt{/}, or \texttt{lastModified} is null.
  \item \textbf{Deduplicate} --- \texttt{dropDuplicates(["model\_id"])}.
\end{enumerate}

Total cleaned records: 2,814,954 (1 record removed).

\section{Output for the Next Project Stage}

The cleaned Hugging Face models dataset will support the following downstream analyses:

\begin{itemize}
  \item \textbf{Adoption signals:} download and like distributions across pipeline tags and libraries to identify which AI tasks and frameworks have the strongest real-world usage.
  \item \textbf{Cross-source join with GH Archive:} linking \texttt{owner/model\_name} to GitHub repository names to correlate GitHub activity (stars, forks, PRs, issues) with Hugging Face adoption metrics.
  \item \textbf{Under-recognized project detection:} models with high downloads but low GitHub stars, or vice versa.
  \item \textbf{Health scoring enrichment:} the \texttt{parameter\_count}, \texttt{license}, \texttt{tag\_count}, and \texttt{downloads} fields will feed into the repository health dashboard.
  \item \textbf{Ecosystem analysis:} comparing model counts, downloads, and community activity across task categories and framework ecosystems.
\end{itemize}
